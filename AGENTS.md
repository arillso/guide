# Arillso Guide

## Context

This repository builds the Arillso documentation site published at
<https://guide.arillso.io>. It is a Sphinx project rendered with
[antsibull-docs](https://github.com/ansible-community/antsibull-docs) and the
`sphinx-ansible-theme`. The build collects role and collection documentation
from the Arillso Ansible collections and renders it into a static HTML site
deployed to GitHub Pages.

This repo contains **no Ansible roles or plugins itself** — it is the
documentation toolchain only.

## Structure

### Build inputs

- `requirements.txt` - Python build dependencies (antsibull-docs, theme, sitemap)
- `conf.py` - Sphinx configuration
- `antsibull-docs.cfg` - antsibull-docs configuration
- `build.sh` - Build entrypoint; renders collection RST and runs Sphinx
- `versions.env` - Pinned versions consumed by the build
- `package.json` - PostCSS tooling for theme CSS (autoprefixer, cssnano)
- `postcss.config.js` - PostCSS configuration

### Source content

- `rst/` - hand-written reStructuredText pages (`guide/`, `github/`,
  `containers/`, `libraries/`); these are the source for the site and are
  edited directly

### Build outputs (git-ignored)

- `rst/collections/` - reStructuredText generated by antsibull-docs from the
  upstream collections
- `temp-rst/` - antsibull-docs scratch output
- `build/` - rendered HTML site (deployed to Pages)

### Static assets

- `_static/` - custom CSS/JS/images
- `_templates/` - Sphinx template overrides
- `CNAME` - custom domain for GitHub Pages
- `.nojekyll` - disables Jekyll processing on Pages

### Container

- `Dockerfile` / `docker-compose.yml` - local build environment

## Conventions

### Code Style

- Use 4 spaces for indentation in YAML and JSON files (see `.editorconfig`)
- Markdown follows `.markdownlint.json`; YAML follows `.yamllint`
- Run hooks locally with `lefthook install`; they run on commit and push

### Building

- Install dependencies: `pip install -r requirements.txt`
- Build the site: `./build.sh`
- Output lands in `build/html`

### Documentation

- Hand-written pages under `rst/` (except `rst/collections/`) are edited
  directly — they are the source for the guide, container, and library docs
- `rst/collections/` is generated from the upstream collections — fix the
  source collection, not the generated output
- Theme and layout changes live in `_static/`, `_templates/`, and `conf.py`

## Workflows

### CI/CD

Event-focused workflows calling reusables from `arillso/.github`:

- **pull-request.yml** (`Pull Request`) - builds the docs, runs the secret
  scan, and runs Claude review on PRs
- **merge.yml** (`Merge to Main`) - builds and deploys to Pages on push to
  `main` and the weekly schedule, plus the secret scan
- **nightly-security.yml** (`Nightly Security Scan`) - scheduled secret scan

## Do Not

- Do not commit secrets or sensitive data
- Do not commit build outputs (`build/`, `temp-rst/`, `rst/collections/`)
- Do not hand-edit generated `rst/collections/` — fix the source collection instead
- Do not unpin build dependency versions without a deliberate update
